### Web Accessibility
- POUR Principles
    - Perceivable: Users must be able to see/hear content.
    - Operable: Users must be navigate/use it.
    - Understandable: Content must be clear and predictable.
    - Robust: Works across devices, browsers and assistive tech.

### Learn WCAG Levels(A, AA, AAA)
- A: Basic Accessibility
- AA: Industry standards
- AAA: Advanced(hard to achieve)

### Need to Focus
- Semantic HTML: 
    - Use proper tags: `<button>, <nav>, <label>`
    - Avoid div soup
- Keyboard Accessibility:
    - Everything should work with `tab/enter/arrow keys`
- Screen Reader Support: 
    - ARIA roles and attributes
    - Test with screen readers(like NVDA or Voiceover)
- Color and Contrast: 
    - Maintain contrast ratio
- Forms Accessibilty
    - Labels, error messages, focus states

### Use Tools like a Pro
- Lighthouse: quick audits
